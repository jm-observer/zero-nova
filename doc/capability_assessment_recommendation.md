# 技术推荐类问题处理能力评估

## 评估目标

评估 zero-nova 当前是否具备处理"技术推荐类"问题的能力（如：推荐最好用的TTS大模型）。

## 问题处理所需能力

| # | 所需能力         | 说明                                     |
|---|-----------------|------------------------------------------|
| 1 | 联网搜索         | 获取最新的产品评测、排行等实时信息           |
| 2 | 网页内容抓取      | 访问搜索结果页面，提取详细对比信息           |
| 3 | 信息整合与推理    | 从多源信息中归纳、对比、给出结构化推荐       |
| 4 | 提示词引导       | 系统提示词中包含处理此类问题的流程指引        |

## 当前能力评估

### 1. 联网搜索 — 有条件具备

- **工具**: `web_search`（`src/tool/builtin/web_search.rs`）
- **实现**: 通过 Brave Search API 进行搜索，支持 query 和 count 参数
- **前提条件**: 需设置 `SEARCH_API_KEY` 环境变量，否则该工具不会注册
- **结论**: 配置 API Key 后可用

### 2. 网页内容抓取 — 已具备

- **工具**: `web_fetch`（`src/tool/builtin/web_fetch.rs`）
- **实现**: 基于 reqwest + scraper，支持 CSS 选择器提取，截断上限 50KB
- **结论**: 可用，无额外配置要求

### 3. 信息整合与推理 — 已具备

- **实现**: Agent 运行时（`src/agent.rs`）支持多轮迭代（最多 `max_iterations` 轮），每轮可并行调用多个工具，LLM 负责推理和整合
- **结论**: 依赖底层 LLM 的推理能力，框架本身不构成瓶颈

### 4. 提示词引导 — 已具备

- **实现**: `prompts/default.md` 已写入技术推荐类问题的处理流程提示词
- **加载方式**: `SystemPromptBuilder` 在编译时通过 `include_str!` 加载 `prompts/default.md` 内容
- **结论**: 系统提示词已包含处理流程

## 综合结论

| 能力项       | 状态              |
|-------------|-------------------|
| 联网搜索     | 需配置 `SEARCH_API_KEY` |
| 网页抓取     | 已就绪             |
| 信息整合推理  | 已就绪             |
| 提示词引导   | 已就绪             |

**zero-nova 已基本具备处理技术推荐类问题的能力。** 唯一的前置条件是配置搜索 API Key。

配置后的完整处理链路：

```
用户提问 → LLM 识别为推荐类问题
        → 调用 web_search 搜索最新评测
        → 调用 web_fetch 抓取详情页
        → LLM 多维度整合
        → 结构化输出推荐结果
```

## 潜在改进点

1. **搜索结果质量**: 当前仅对接 Brave Search，可考虑通过 MCP 扩展更多搜索源
2. **max_iterations 上限**: CLI 默认为 5 轮，复杂推荐场景可能需要更多轮次（搜索 + 多次抓取 + 整合）
3. **max_tokens 限制**: CLI 默认 1024 tokens，结构化推荐输出可能需要更大的输出长度
4. **缓存机制**: 无搜索结果缓存，重复提问会重复调用搜索 API，可考虑加入短期缓存
