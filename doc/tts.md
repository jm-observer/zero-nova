# TTS 高级情感控制实验方案 (DGX Spark 版)

## 1. 核心目标
利用 DGX Spark 的高性能计算能力，通过 **GPT-SoVITS** 实现对“魅惑”、“俏皮”、“疲惫”等细分情感的深度定制与高保真还原。

## 2. 实验架构设计

### 2.1 环境层 (Environment)
- **硬件驱动**: NVIDIA Driver + CUDA 12.x + cuDNN
- **容器化方案**: 基于 `nvcr.io/nvidia/pytorch` 镜像构建 Docker 容器
- **关键依赖**: 
  - `GPT-SoVITS` (核心模型)
  - `FFmpeg` (音频编解码)
  - `Librosa` (情感特征提取与分析)
  - `Flash Attention 2` (加速训练，利用 A100/H100 特性)

### 2.2 数据层 (Data Pipeline)
- **数据结构**: 采用 `[音频文件] + [文本内容] + [情感标签]` 的三元组结构。
- **自动化预处理**:
  - 使用 **Whisper** 进行高精度语音转文字 (ASR)。
  - 使用 **LLM (如 GPT-4o)** 对文本进行情感语义分析，生成 Prompt 标签。
  - 使用 **Librosa** 提取音频的 Pitch (音高) 和 Energy (能量) 曲线，辅助验证情感的一致性。

### 2.3 模型层 (Model Strategy)
- **阶段一：基础训练 (Pre-training)**
  - 在大规模通用语音数据集上进行基础训练。
- **阶段二：情感微调 (Emotional Fine-tuning)**
  - **策略 A: LoRA (Low-Rank Adaptation)** - 针对特定情感（如“魅惑”）训练轻量化插件，保持基础音色不变，仅改变情感特征。
  - **策略 B: Full Fine-tuning** - 利用 DGX 的显存优势，进行全参数微调，追求极致的情感表现力。

## 3. 情感实现路径对照表

| 目标情感 | 音频特征要求 (Reference Audio) | 文本诱导策略 (Text Prompting) |
| :--- | :--- | :--- |
| **魅惑 (Flirtatious)** | 低音调、高气声、语速缓慢、强停顿 | 使用暗示性词汇，增加“...”和长停顿 |
| **俏皮 (Playful)** | 高音调、语速轻快、能量波动大 | 使用感叹号、语气词（嘿嘿、喔~） |
| **疲惫 (Tired)** | 音量减弱、音调低沉、语速极慢 | 使用省略号、语气词（唉、嗯...） |
