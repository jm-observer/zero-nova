# 2026-04-21 UI 优化与录音快捷键支持

## 时间
2026-04-21

## 项目现状
1.  **快捷键触发录音**：目前应用内缺乏直接触发外部软件录音功能的逻辑。当窗口处于焦点时，普通的 `Ctrl+Win` 有时无法穿透。

## 本次目标
1.  **快捷键透传**：在应用内检测并模拟转发 `Ctrl+Win`，确保能调起外部录音软件。

## 详细设计

### 1. 快捷键透传方案
*   **前端监听 (`ChatView.ts`)**:
    *   在 `document` 级别监听 `keydown` 事件。
    *   主动检测 `Ctrl + Win` 组合键。当检测到按键时，调用后端模拟指令。
*   **后端模拟 (`system.rs`)**:
    *   底层引入 `enigo` 库用于系统级按键模拟。
    *   实现 `trigger_recording_hotkey` 指令，执行 `KeyDown(Control)` + `KeyDown(Meta)` 并随即释放。
    *   **原理**：这种主动补发转发机制解决了当前窗口处于焦点时可能干扰后台软件捕获按键的问题。

## 测试案例
1.  **组合键透传**：在输入框聚焦状态下按下 `Ctrl + Win`，验证后台录音软件是否能正常捕获并启动。

## 风险与待定项
*   **模拟权限**：若目标软件运行在更高权限（管理员模式），模拟按键可能无效。
> [!CAUTION]
> **当前方案存在的问题 (2026-04-21 追记)**:
> 目前通过 `enigo` 模拟 `Ctrl + Win` 的方案在部分环境下会导致 Windows 键被独立触发（弹出开始菜单），未能稳定达成预期的按键透传。
> 已决定暂停代码修改，后续将考虑更底层的按键Hook或调整模拟时序。
