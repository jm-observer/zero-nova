# 任务跟进：前端如何感知后端运行状态

## 时间
2026-04-20

## 项目现状
目前在执行自动化 Benchmark 流程时，用户看到的是一段描述性的文本（"🚀 任务启动中..."），无法直观、实时地通过前端界面感知后端的具体进度或运行状态。

## 本次目标
研究并设计一套机制，使得前端界面能够实时展示后端任务的运行状态（例如：正在运行、进度百分比、当前步骤、已完成任务数等），而不是仅仅显示一段静态的等待提示。

## 详细设计
待调研方案：
1. **轮询 (Polling)**: 前端定期请求后端 API 获取当前任务状态。
2. **WebSocket**: 后端主动向前端推送状态变更，实现实时通信。
3. **Server-Sent Events (SSE)**: 后端单向向前端推送任务进度流。
4. **日志流 (Log Streaming)**: 将后端的 stdout/stderr 通过特定通道实时流式传输至前端。

## 测试案例
- 任务启动时，界面显示“启动中”。
- 任务执行过程中，进度条或状态描述随后端步骤实时更新。
- 任务完成时，界面立即跳转或显示“完成”及结果链接。
- 任务异常中断时，界面能显示具体的错误信息。

## 风险与待定项
- 需要评估当前技术栈对 WebSocket 或 SSE 的支持程度。
- 实时通信对后端资源的消耗情况。
