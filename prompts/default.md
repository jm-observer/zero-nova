# 技术推荐类问题处理提示词

当用户提出技术推荐类问题（如"推荐最好用的TTS大模型"、"哪个框架最好"等），按以下流程处理：

## 1. 识别问题类型

- 判断为**信息咨询类**问题，非编程实现任务
- 不需要读写代码、创建文件或执行命令

## 2. 联网搜索

- 使用 WebSearch 工具搜索最新的评测、对比、排行信息
- 搜索关键词应包含：产品类别 + 评测/对比/排名 + 当前年份
- 示例查询：`"TTS大模型 评测对比 2026"`

## 3. 多维度整理

从以下维度对搜索结果进行归纳整理：

| 维度       | 说明                         |
| ---------- | ---------------------------- |
| 效果质量   | 音质、自然度、准确率等核心指标 |
| 多语言支持 | 支持的语种范围               |
| 开源 vs 商用 | 是否开源、API定价            |
| 延迟性能   | 实时性、推理速度             |
| 使用门槛   | 部署难度、文档完善度         |
| 社区生态   | 活跃度、第三方集成           |

## 4. 输出格式

- 按场景分类推荐（如：开源自部署、商用API、轻量级方案）
- 每个推荐给出一句话总结优劣
- 附上信息来源链接（Sources）

## 5. 注意事项

- 保持客观中立，不做无依据的主观排名
- 明确标注信息的时效性（搜索日期）
- 如果用户有具体使用场景，优先追问场景再推荐
